2//! # Usage
3//! Generally, there are two types of union supported by this library.
4//! 1. Union by rank: useful if you want to use a union find simply as a
5//! disjoint set with logarithmic lookup time
6//! 2. Custom unioning: likely what you want when you are, for example, implementing a type checker.
7//!
8//! This library abstracts over the backing storage of the union find. There are three options
9//! 1. [`Vec`]: very fast lookups as long as keys are usize. Other keys are not supported.
10//! 2. [`HashMap`]: works for any key that implements [`Hash`](std::hash::Hash) and [`Eq`]
11//! 3. [`BTreeMap`]: works for any key that implements [`Ord`]
12//!
13//! Internally, a union find contains one ore more of these datastructures. One if
14//! you use custom unioning, and two if you use union by rank (the ranks are stored separately).
15//! However, even with custom unioning you could have a second HashMap for custom additional
16//! information stored alongside the union find keys, similar to how union by rank works.
17//!
18//! Path shortening is optionally performed during find operations by calling
19//! [`find_shorten`](UnionFind::find_shorten) instead of [`find`](UnionFind::find).
20//! By using [`find_shorten`](UnionFind::find_shorten), subsequent finds become faster than the first.
21//! However, an advantage to [`find`](UnionFind::find) is that it does not need mutable access to the datastructure
22
