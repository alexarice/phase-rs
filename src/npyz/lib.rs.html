3/*!
4Serialize and deserialize the NumPy's
5[*.npy binary format](https://docs.scipy.org/doc/numpy-dev/neps/npy-format.html).
6
7# Overview
8
9[**NPY**](https://docs.scipy.org/doc/numpy-dev/neps/npy-format.html) is a simple binary data format.
10It stores the type, shape and endianness information in a header,
11which is followed by a flat binary data field. This crate offers a simple, mostly type-safe way to
12read and write *.npy files. Files are handled using iterators, so they don't need to fit in memory.
13
14## Optional cargo features
15
16No features are enabled by default.  Here is the list of existing features:
17
18* There are a couple of features which enable support for serialization/deserialization of foreign
19  types. These require opt-in because they can be stability hazards; a major version bump of `npyz`
20  may introduce a major version bump of one of these crates.  (NOTE: to ease this issue somewhat,
21  `npyz` will re-export the versions of the crates it uses)
22  * **`"complex"`** enables the use of [`num_complex::Complex`].
23  * **`"half"`** enables the use of [`half::f16`].
24  * **`"arrayvec"`** enables the use of [`arrayvec::ArrayVec`] and [`arrayvec::ArrayString`]
25    as alternatives to `Vec` and `String` for some string types.
26* **`"derive"`** enables derives of traits for working with structured arrays.
27* **`"npz"`** enables adapters for working with NPZ files
28  (including scipy sparse matrices),
29  adding a public dependency on the `zip` crate.
30  This requires opt-in because `zip` has a fair number of transitive dependencies.
31  (note that some npz-related helper functions are available even without the feature)
32
33## Reading
34
35Let's create a simple *.npy file in Python:
36
37```python
38import numpy as np
39a = np.array([1, 3.5, -6, 2.3])
40np.save('test-data/plain.npy', a)
41```
42
43Now, we can load it in Rust using [`NpyFile`]:
44
45```rust
46fn main() -> Result<(), Box<dyn std::error::Error>> {
47    let bytes = std::fs::read("test-data/plain.npy")?;
48
49    // Note: In addition to byte slices, this accepts any io::Read
50    let npy = npyz::NpyFile::new(&bytes[..])?;
51    for number in npy.data::<f64>()? {
52        let number = number?;
53        eprintln!("{}", number);
54    }
55    Ok(())
56}
57```
58
59And we can see our data:
60
61```text
621
633.5
64-6
652.3
66```

68### Inspecting properties of the array
69
70[`NpyFile`] provides methods that let you inspect the array.
71
72```rust
73fn main() -> std::io::Result<()> {
74    let bytes = std::fs::read("test-data/c-order.npy")?;
75
76    let data = npyz::NpyFile::new(&bytes[..])?;
77    assert_eq!(data.shape(), &[2, 3, 4]);
78    assert_eq!(data.order(), npyz::Order::C);
79    assert_eq!(data.strides(), &[12, 4, 1]);
80
81    // convenience method for reading to vec
82    println!("{:?}", data.into_vec::<f64>());
83    Ok(())
84}
85```

87## Writing
88
89The primary interface for writing npy files is the [`WriterBuilder`] trait.
90
91```rust
92use npyz::WriterBuilder;
93
94fn main() -> std::io::Result<()> {
95    // Any io::Write is supported.  For this example we'll
96    // use Vec<u8> to serialize in-memory.
97    let mut out_buf = vec![];
98    let mut writer = {
99        npyz::WriteOptions::new()
100            .default_dtype()
101            .shape(&[2, 3])
102            .writer(&mut out_buf)
103            .begin_nd()?
104    };
105
106    writer.push(&100)?;
107    writer.push(&101)?;
108    writer.push(&102)?;
109    // you can also write multiple items at once
110    writer.extend(vec![200, 201, 202])?;
111    writer.finish()?;
112
113    eprintln!("{:02x?}", out_buf);
114    Ok(())
115}
116```

123## Working with `ndarray`
124
125Using the [`ndarray`](https://docs.rs/ndarray) crate?  No problem!
126At the time, no conversion API is provided by `npyz`, but one can easily be written:
127
128```rust
129// Example of parsing to an array with fixed NDIM.
130fn to_array_3<T>(data: Vec<T>, shape: Vec<u64>, order: npyz::Order) -> ndarray::Array3<T> {
131    use ndarray::ShapeBuilder;
132
133    let shape = match shape[..] {
134        [i1, i2, i3] => [i1 as usize, i2 as usize, i3 as usize],
135        _  => panic!("expected 3D array"),
136    };
137    let true_shape = shape.set_f(order == npyz::Order::Fortran);
138
139    ndarray::Array3::from_shape_vec(true_shape, data)
140        .unwrap_or_else(|e| panic!("shape error: {}", e))
141}
142
143// Example of parsing to an array with dynamic NDIM.
144fn to_array_d<T>(data: Vec<T>, shape: Vec<u64>, order: npyz::Order) -> ndarray::ArrayD<T> {
145    use ndarray::ShapeBuilder;
146
147    let shape = shape.into_iter().map(|x| x as usize).collect::<Vec<_>>();
148    let true_shape = shape.set_f(order == npyz::Order::Fortran);
149
150    ndarray::ArrayD::from_shape_vec(true_shape, data)
151        .unwrap_or_else(|e| panic!("shape error: {}", e))
152}
153
154pub fn main() -> std::io::Result<()> {
155    let bytes = std::fs::read("test-data/c-order.npy")?;
156    let reader = npyz::NpyFile::new(&bytes[..])?;
157    let shape = reader.shape().to_vec();
158    let order = reader.order();
159    let data = reader.into_vec::<i64>()?;
160
161    println!("{:?}", to_array_3(data.clone(), shape.clone(), order));
162    println!("{:?}", to_array_d(data.clone(), shape.clone(), order));
163    Ok(())
164}
165```
166
167Likewise, here is a function that can be used to write an ndarray:
168
169```rust
170use std::io;
171use std::fs::File;
172
173use ndarray::Array;
174use npyz::WriterBuilder;
175
176// Example of writing an array with unknown shape.  The output is always C-order.
177fn write_array<T, S, D>(writer: impl io::Write, array: &ndarray::ArrayBase<S, D>) -> io::Result<()>
178where
179    T: Clone + npyz::AutoSerialize,
180    S: ndarray::Data<Elem=T>,
181    D: ndarray::Dimension,
182{
183    let shape = array.shape().iter().map(|&x| x as u64).collect::<Vec<_>>();
184    let c_order_items = array.iter();
185
186    let mut writer = npyz::WriteOptions::new().default_dtype().shape(&shape).writer(writer).begin_nd()?;
187    writer.extend(c_order_items)?;
188    writer.finish()
189}
190
191pub fn main() -> io::Result<()> {
192    let array = Array::from_shape_fn((6, 7, 8), |(i, j, k)| 100*i as i32 + 10*j as i32 + k as i32);
193    // even weirdly-ordered axes and non-contiguous arrays are fine
194    let view = array.view(); // shape (6, 7, 8), C-order
195    let view = view.reversed_axes(); // shape (8, 7, 6), fortran order
196    let view = view.slice(ndarray::s![.., .., ..;2]); // shape (8, 7, 3), non-contiguous
197    assert_eq!(view.shape(), &[8, 7, 3]);
198
199    let mut file = io::BufWriter::new(File::create("examples/output/ndarray.npy")?);
200    write_array(&mut file, &view)
201}
202```

204## Structured arrays
205
206`npyz` supports structured arrays!  Consider the following structured array created in Python:
207
208```python
209import numpy as np
210a = np.array([(1,2.5,4), (2,3.1,5)], dtype=[('a', 'i4'),('b', 'f4'),('c', 'i8')])
211np.save('test-data/simple.npy', a)
212```
213
214To load this in Rust, we need to create a corresponding struct.
215There are three derivable traits we can define for it:
216
217* [`Deserialize`] — Enables easy reading of `.npy` files.
218* [`AutoSerialize`] — Enables easy writing of `.npy` files. (in a default format)
219* [`Serialize`] — Supertrait of `AutoSerialize` that allows one to specify a custom [`DType`].
220
221**Enable the `"derive"` feature in `Cargo.toml`,**
222and make sure the field names and types all match up:
223*/
224
225// It is not currently possible in Cargo.toml to specify that an optional dependency should
226// also be a dev-dependency.  Therefore, we discretely remove this example when generating
227// doctests, so that:
228//    - It always appears in documentation (`cargo doc`)
229//    - It is only tested when the feature is present (`cargo test --features derive`)
230#![cfg_attr(any(not(doctest), feature="derive"), doc = r##"
231```
232// make sure to add `features = ["derive"]` in Cargo.toml!
233#[derive(npyz::Deserialize, Debug)]
234struct Struct {
235    a: i32,
236    b: f32,
237    c: i64,
238}
239
240fn main() -> Result<(), Box<dyn std::error::Error>> {
241    let bytes = std::fs::read("test-data/structured.npy")?;
242
243    let npy = npyz::NpyFile::new(&bytes[..])?;
244    for row in npy.data::<Struct>()? {
245        let row = row?;
246        eprintln!("{:?}", row);
247    }
248    Ok(())
249}
250```
251"##)]
252/*!
253The output is:
254
255```text
256Array { a: 1, b: 2.5, c: 4 }
257Array { a: 2, b: 3.1, c: 5 }
258```
